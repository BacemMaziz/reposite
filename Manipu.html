Manipulation

Manipulation du projet en Mode Console et Mode Graphique

1. Mode Console

Le mode console de l'application permet à l'utilisateur d'interagir avec le programme via une interface en ligne de commande. Ce mode est principalement destiné aux utilisateurs qui préfèrent une interface plus simple ou à des fins de test et de développement.

Fonctionnalités du Mode Console :

  • Lancement de l'application : Le programme se lance directement dans un terminal ou une fenêtre de console. L'utilisateur peut entrer des commandes pour exécuter différentes actions.
  • Navigation via commandes :
    • ajouter-image [chemin] : Permet d'ajouter une image au système pour traitement.
    • extraire-metadata [image] : Extrait les métadonnées d'une image spécifique.
    • comparaison-dossiers : Compare les dossiers spécifiés et affiche les différences dans les images ou métadonnées.
  • Retour de résultats : Les résultats des commandes seront affichés directement dans la console.
  • Interaction simplifiée : L'utilisateur est guidé par des messages explicites sur les actions possibles et les erreurs potentielles.

2. Mode Graphique (GUI)

Le mode graphique fournit une interface utilisateur (UI) visuelle à l'aide de Swing, permettant une interaction plus intuitive et agréable pour l'utilisateur. Ce mode est recommandé pour les utilisateurs qui préfèrent une navigation plus visuelle.

Figure 1: mode graphique capture

Fonctionnalités du Mode Graphique :

  • Interface de navigation : Une fenêtre principale avec des menus et des boutons permettant de sélectionner des actions comme ajouter des images, visualiser des métadonnées ou comparer des dossiers.
  • Ajout d'images : Un bouton permet de sélectionner un fichier image depuis l'ordinateur et d'afficher les informations pertinentes dans un tableau ou une fenêtre de métadonnées.
  • Affichage des métadonnées : Les métadonnées extraites des images sont présentées dans une fenêtre dédiée sous forme de tableau ou de liste.
  • Comparaison de dossiers : Un bouton permet de choisir deux dossiers à comparer. Les différences entre les images seront affichées de manière graphique.
  • Accessibilité : La navigation se fait par des boutons interactifs et des fenêtres de dialogue, rendant l'application accessible à tous types d'utilisateurs.

Avantages et utilisations de chaque mode

Mode Console

  • Avantages : Légèreté, utilisation rapide pour les développeurs ou dans des environnements de production.
  • Utilisation : Test de fonctionnalités, automatisation de tâches via des scripts, ou utilisateurs techniques préférant la ligne de commande.

Mode Graphique

  • Avantages : Interface plus conviviale et facile d'accès pour les utilisateurs finaux.
  • Utilisation : Idéal pour les utilisateurs non techniques, pour la présentation de résultats ou la gestion facile de l'application.

Conclusion

Le choix entre les modes console et graphique dépend de l'utilisateur et de ses préférences. L'application est conçue pour être flexible et permet à l'utilisateur de choisir le mode le plus adapté à ses besoins.
